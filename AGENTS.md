# AGENTS

## Общие правила проекта

- Код является источником правды.
- Перед крупными правками коротко определить затронутые системы и план.
- Держать изменения узкими: не делать unrelated refactor рядом с фичей.
- Новые игровые системы дробить по смыслу: core loop, scene, UI, simulation, actions, models, data.
- Максимальная длина одного C#-скрипта: 500 строк. Если файл приближается к лимиту, разбить его на несколько `partial`/сервисных файлов до продолжения работы.
- После добавления, удаления или переименования `.cs` файлов обновлять `Assembly-CSharp.csproj`.
- Для русских UI-текстов сохранять UTF-8 и проверять отсутствие mojibake.
- После изменений кода запускать `dotnet build Assembly-CSharp.csproj -v:minimal`.
